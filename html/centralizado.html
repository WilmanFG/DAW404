Sistemas de control de versiones Centralizados


Los sistemas de control de versiones centralizados se desarrollaron para registrar los cambios en un sistema central y permitir a los desarrolladores colaborar en otros sistemas. Los sistemas de control de versiones centralizados tienen mucho que ofrecer, sin embargo, también tiene algunas desventajas serias.

Ventajas

Relativamente fácil de configurar

Aporta transparencia

Permitir que los administradores controlen el flujo de trabajo

Desventajas

Si el servidor principal deja de funcionar, los desarrolladores no pueden guardar los cambios versionados

Las confirmaciones remotas aveces son lentas

Los cambios no solicitados pueden arruinar el desarrollo

Si la base de datos central está dañada, se podría perder todo el historial (problemas de seguridad)
